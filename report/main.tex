\begin{abstract}
    \noindent In the following we implement Lanczos and Arnoldi algorithm to study the problem of
    matrix functions evaluation in the context of graph signal processing. We begin studying and
    assessing the computational cost of these algorithms and their orthogonalization variants. Then, through a series of
    experiments, we consider the iteration residual, to relate it with the real error and to study
    how it affects convergence.
\end{abstract}

\section{Introduction}

We introduce basic graph theory concepts and we recall some theoretical results for the experiment.

\subsection{Signal processing on graphs}



We consider a weighted undirected graph $ \G = (\V, \E, \mathcal{W}) $,
where $ \V \subset \R^N $ is the set of vertices, $ \E \subset \R^M $ is the set of edges, and $ \mathcal{W}
: \V \times \mathcal{V} \to \R$ is a weight function. We represent the weight function with a $ N \times N $ matrix $ W $ such that 
$W_{i,j} = 
\begin{cases} 
    W(v_i, v_j), & \text{if } (v_i, v_j) \in \E \\ 
    0, & \text{otherwise} 
\end{cases} 
\quad \text{for all } i,j = 1, \dots, |\V|
$,
and for our needs, we assume $ W $ to be symmetric, that is $ W_{i,j} =  W_{j,i} $.


\begin{wrapfigure}{r}{6.5cm}
  \caption{Filtering signal on graphs.} 
  \label{fig:sigproc}
  \includegraphics[width=7.5cm]{assets/plots/printed_graphs.pdf}
\end{wrapfigure}

In the study of graph signal processing, we model the idea of sending a signal to a node as
assigning a value to a vertex with a function $ s : \V \to \R$, whose values can be represented as a vector
$ s = s(\V) = [s_1, \dots, s_N] \in \R^N $ where each entry $ s_i \defeq s(v_i)$ represents
the signal sent over a node $ v_i \in \V$. We also keep track of the weight of each vertex
with a function $ d(i) \defeq \sum_{j=1}^N W_{i,j} $, and we call $D =
\text{diag}(d(1), \dots, d(N))$ the degree matrix.

In this setting, we introduce the graph Laplacian $\L$ defined as 
$ \L = D - W$. By
construction, $\L^T = \mathcal{L}$, thus the Lanczos method can be safely applied to our 
problem. This matrix has some other useful properties that we will list in the following Proposition.
\begin{proposition}
  The Laplacian matrix $ \L \in \R^{N\times N} $ satisfies the following properties:
  \begin{enumerate}
    \item $ \L $ is symmetric and positive semi-definite.
    \item The smallest eigenvalue of $ \L $ is $ 0 $ with corresponding eigenvector the constant one
      vector $ \mathbf{1} $.
    \item $ \L $ has $ N $ non-negative real-valued eigenvalues $ 0 = \lambda_0 \le \lambda_1 \le
      \ldots \le \lambda_{N-1}$.
  \end{enumerate}
\end{proposition}
\begin{proof}
  See \cite{von2007tutorial}.
\end{proof}


\begin{remark}
  In light of this proposition and by the spectral theorem, the Laplacian can be decomposed as:
  \begin{equation*}
    \L = U \Lambda U^{*},
  \end{equation*}
  where $ U = \left[ u_0,\dots,u_{N-1} \right]$ is a $ N\times N $ orthogonal matrix and we call it \emph{Fourier basis}, and $ \Lambda =
  \text{diag}\left(\lambda_0, \dots, \lambda_{N-1}\right)$ is the matrix of its eigenvalues, which
  are monotonically increasing, and the vectors in $ U $ are in the same order that the eigenvalues.
  \end{remark}

\begin{definition}[Graph signal]
  A graph signal is a continuous function $ g : \R^+ \to \R $. For our needs, we will assume $ g $ to be
  defined on the spectrum of $ \L $.
\end{definition}
Diagonalising the Laplacian would give an easy way to compute the function $ g(\L) $ by
evaluating it on the eigenvalues of $ \L $, which means computing $ g(\L) = U g(\Lambda) U^{*} $.
In matrix notation, applying a filter to a graph $ \G $ corresponds to the operation
\begin{equation*}
  s^\prime \defeq g(\L)s = U g(\L) U^{*}s.
\end{equation*}
However, computing the Fourier basis for $ \L $ is computationally expensive and unstable for large
graphs. This motivates the idea for using the Lanczos method for evaluating the matrix function $ g(\L)$.

In this work, we will restrict our analysis to two graph models: Erdős-Rényi and Sensor graphs. We shall
define them as the current implementation in Library \cite{pygsp}.
\begin{definition}[Erdős-Rényi graph]
  \label{def:ER}
  We define the Erdős-Rényi graph $ \G_{N,p} $ with $ N $ nodes, and probability $ p $, as the graph
  constructed by randomly connecting two nodes independently from any other edge, with probability $ p $.
\end{definition}
\begin{definition}[Sensor graph]
  \label{def:Sensor}
  A sensor graph $ \G_N $ is the graph constructed by picking $ N $ points on the square $ \left[
  0,1 \right] \times \left[ 0,1 \right] $ and connecting points with edges using $ k $ nearest
  neighbours.
\end{definition}
\begin{remark}
  As currently stating in \href{https://github.com/epfl-lts2/pygsp/blob/4716b12c470c579e3f477f2d1c1d99752c73252d/pygsp/graphs/nngraphs/sensor.py#L6}{PyGSP
    sensor graph source code}, definition of Sensor graph changed in 2019, switching to model
    generation using k-means algorithm. This might give slightly different results than the 2015's article
    \cite{susnjara2015} we are trying to reproduce, but we don't investigate this difference.
  
\end{remark}



\subsection{The Lanczos method}

\begin{definition}[Krylov subspace]
 Given a matrix $ A \in \R^{N \times N} $ and a vector $ b \in R^N $, the Krylov subspace of order
 $l$ is defined as the set $ \mathcal{K}_l (A,b) = \text{Span} \left(b, Ab, A^2b, \dots,
A^{l-1}b\right)  $. We represent the basis of this subspace with a matrix $ V_M = \left[ v_1,\dots,v_M
\right] \in \R^{N \times M}$.
\end{definition}
From the Arnoldi relation,
\begin{equation*}
  AV_l = V_lH_l + h_{l+1,l}v_{l+1}e_l^{*}, \\ \hspace{20pt}
  H_l = V_l^{*}AV_l =
  \begin{bmatrix}
    h_{11} & \dots & \dots & h_{1,l} \\
    h_{21} & h_{22} & & \vdots \\
           & \ddots & \ddots & \vdots \\
           & & h_{l,l-1} & h_{l,l}.
  \end{bmatrix},
\end{equation*}
we can derive the Lanczos method for a symmetric, positively defined matrix $ A $. Setting
\begin{equation*}
\alpha_l \defeq h_{l,l},\hspace{20pt}\beta_l \defeq h_{l+1,l}.
\end{equation*}
From our hypothesis on $ A $, it follows that the matrix $ H_l $, which is both upper and lower Hessenberg 
matrix, is also symmetric; in other words, it is tridiagonal---thus we rename it $ T_l $. Hence
the Arnoldi relation takes the form:

\begin{equation*}
  AV_l = V_l \alpha_l  + \beta_l v_{l+1}e_l^{T}, \\ \hspace{20pt}
  T_l = V_l^{\top} A V_l = \begin{bmatrix}
    \alpha_1 & \beta_1 \\
    \beta_1 & \ddots & \ddots \\
      & \ddots & \ddots & \beta_{l-1} \\
      & & \beta_{l-1} & \alpha_l
  \end{bmatrix}.
\end{equation*}.

We now introduce an useful result for evaluating a matrix function using the Arnoldi projection.
\begin{proposition}
 Consider the filter $ g : [0, \lambda_{\text{max}}] \to \R$ and a signal vector $ s \in \R^N $.
 Then the Lanczos method approximates $ g(\L)s $ with: \footnote{This results holds in general for any function $f :\Omega \to \C$, where $ \Omega \subset \Lambda(A) $.}
\begin{equation}
 g(\L)s \approx  V_M \norm{s}_2 g(T_M) e_1 \eqdef g_M. \label{eq:projection}
\end{equation}
\end{proposition}
Numerically, we often compute the eigendecomposition $ (\Lambda,U) $ from $ T_M $, and then
substitute $ g(T_M) = Ug(\Lambda)U^{\top} $ in the equation above.


\subsection{Algorithm}
\label{ssec:algo}

We begin
by defining some quantities that will be useful for describing our implementation of the algorithms.

\begin{definition}(Errors)
  \label{def:errors}
  We define the Lanczos \emph{iteration error} as $\norm{g_{M+j} - g_M}_2 $, where $ j $ is small,
  and the \emph{true error} as $ \norm{e_M}_2 = \norm{g(\L)s - g_M}_2 $.
\end{definition}
We also define error bounds:
\begin{definition}
  We indicate with $ \tau \eqdef \tau_{\text{STOP}} $ the algorithm stopping threshold, that is when
  $ \| g_{M+3} -g_M \|_2< \tau $, and with $ \delta \eqdef \delta_{\text{ORTH}} $ the orthogonality
  threshold, that is when $ \| V^{\top} V - I \|_2 < \delta $.
\end{definition}



In floating point arithmetic the basis $ V_M $ suffers from loss of orthogonality, and orthogonalizing
the incoming vector $ w $ only against $ v_j,v_{j-1} $ in insufficient especially when the input
matrix $ A $ is badly conditioned. Hence, the need to
reorthogonalize the new vector $ w $ against all the previous basis vectors using some steps of the
Grahm-Schmidt process. To do so, we implement three algorithms, which we refer as Lanczos in
Algorithm \ref{alg:lanczos} and Arnoldi in Algorithm \ref{alg:arnoldi_double_ortho}, which we
implement following \cite{demmel1997} and \cite{susnjara2015}. These two algorithms are often called
by the orthogonalization selection in Algorithm \ref{alg:orthosel}, and include a stopping criteria
based on matrix function convergence. Before describing the algorithms, we explain how $ g_M $ is
evaluated in the lanczos process using Equation \ref{eq:projection}.
Recall that $  g_M \defeq \|b \|_2V_Mg(T_M)e_1
$, where $ g(T_M) $ is computed by diagonalizing $ T_M $: because $ T_M $ is symmetric, real and tridiagonal, 
computing its eigendecomposition is stable and efficient; indeed the structure of $ T_M $ allows us
to use a divide and conquer algorithm for eigenvalues, and we'll use \texttt{eigh\_tridiagonal}
routine from Scipy which has cost $ O(N^2) $.

\begin{algorithm}[ht!]
  \caption{Modified Lanczos method in floating point arithmetic.}
  \label{alg:lanczos}
  \begin{algorithmic}
    \State \textbf{Input:} Symmetric matrix $ A \in \R^{N\times N}$, vector $ b \in R^N $, $ M \in
    \N $, function $ g $, tolerance $ \tau $.
    \State \textbf{Output: } Basis matrix $ V_l = \left[ v_1,\dots,v_l \right] \in R^{N \times l} $ for $ K_l(A,b) $, vectors $
    \alpha =\left\{ \alpha_1,\dots,\alpha_l \right\} \in \R^l, \beta=\left[ \beta_1,\dots,\beta_{l-1} \right] \in \R^{l-1} $, where $ l \in
    \{ 1,\dots,M \}$ is the stopping iteration index.
    \State Initialise $ V \leftarrow \mathbf{0}^{N \times M} $ , $ \alpha \leftarrow \mathbf{0}^N $,
    and $ \beta \leftarrow \mathbf{0}^{N-1}$

    \For{$ j = 1, \dots, M-1 $}
    \State $ w \leftarrow Av_j  $
    \State $ \alpha_j \leftarrow v_j^{\top} w $

    \State $\left\{
    \begin{array}{ll}
        w \leftarrow w - \alpha_j v_j&  \text{No reorthogonalization} \\
        \text{\textbf{if} } j>0 \text{ \textbf{then} } w \leftarrow w - \beta_j v_{j-1}\text{ \textbf{end
    if}} & \\ \\
        w = w - \sum_{i=1}^j (w^{\top}v_i)v_i & \text{Full reorthogonalization} \\
        w = w - \sum_{i=1}^j (w^{\top}v_i)v_i 
    \end{array}
    \right.$
    \If{$ j < M $}
    \State $ \beta_j = \|w\|_2 $
    \If{$ \beta_j = 0 $}
      \State 
      \Return $ V_j, \alpha_{1:j}, \beta_{1:j-1} $
      \Comment{--- Breakdown ---}
    \Else
      \If{$ \| g_j - g_{j-3}\|_2  < \tau$}
      \State 
      \Return $ V_j, \alpha, \beta $
      \Comment{--- Breakdown ---}
      \EndIf
      \State $ v_{j+1} = \frac{w}{\beta_j} $
    \EndIf
    \EndIf
    \EndFor
    \State \Return $ V_M, \alpha_{1:M}, \beta_{1:M-1} $.
  \end{algorithmic}
  
\end{algorithm}

\begin{algorithm}[ht!]
  \caption{Approximation of $ g(\L)s $ using Lanczos.}
  \label{alg:g_evaluation}
  \begin{algorithmic}
    \State \textbf{Input:} Function $ g $, Matrix $ T $, signal $ s $.
    \State $ \lambda, U \leftarrow $ diagonalize($ T $)  \Comment{Here $ \lambda $ is the vector of
    eigenvalues.}
    \State $ u_1 \leftarrow  (U^{\top})_{:,1}$
    \State $y \leftarrow \|s\| U (\text{Diag}(g(\lambda)) U^{\top})e_1 = \|s\| U (g(\lambda) \odot u_1)$
    \State \Comment{Here $ u \odot v $ denotes the element-wise product between
  two vectors $ u $ and $ v $.}
    \State \Return $ Vy $
  \end{algorithmic}
\end{algorithm}

\begin{algorithm}[ht!]
  \caption{Evaluation of $ g(\L)s $ using Fourier basis.}
  \label{alg:g_evaluation_fourier}
  \begin{algorithmic}
    \State \textbf{Input:} Function $ g $, Graph $ \G $, signal $ s $.
    \State $ \Lambda,U \leftarrow \text{Fourier\_basis}( \G )$
    \State \Return $ U g(\Lambda) U^{\top}s $
  \end{algorithmic}
  
\end{algorithm}

\begin{algorithm}[ht!]
  \caption{Orthogonalization selection.}
  \label{alg:orthosel}
  \begin{algorithmic}
    \State \textbf{Input: $\delta = 10^{-10}$}.
    \State $V, T, j \leftarrow $ Lanczos(A,b, classic)
    \If {$ \|V^{\top} V - I\|_2 < \delta $}
    \State $ V, T,j \leftarrow $ Lanczos(A,b, full)
      \EndIf
  \end{algorithmic}
\end{algorithm}




\subsubsection{Computational cost}
\label{ssec:cost}

Here we overview computational cost of the algorithm that we tested, for an input sparse matrix $ A $,
where $ \text{nnz}(A) $ is the number of non-zero elements of $ A $:

\begin{itemize}
  \item \textbf{Lanczos:} Computes $ V $  with partial orthogonalization, with $ O(M  N
  + M \cdot \text{nnz}(A)) $. Note that the product $ Av_j $ dominates the cost,
  and that $ \text{nnz}(A) = N^2 $ when $ A $ is dense. When we need to evaluate function $ g $ at
  each iteration, such as in Section \ref{ssec:ex23}, we must consider also the $ O(N^2) $
  operations to compute $ g(\L) $ with divide and conquer, thus the total cost in this case becomes $ O(M N^2) $.
\item \textbf{Lanczos algorithm with double orthogonalization:} It differs from Lanczos by the
  computation of $ w = w -
  \sum_{i=1}^j v_i(v_i^{\top}w) $, which can cost up
  to $ O(M N) $. Thus the total cost becomes $ O(M^2N +  M\cdot \text{nnz}(A)) $, and when we
  also evaluate $ g $ it becomes $ O(M^2N + MN^2) $.
\item \textbf{Arnoldi with double orthogonalization}: For the orthogonalization it takes $ O(M^2N)
  $, plus the matrix vector product at each step with $ A $, hence again $ O(M^2N + M \cdot
  \text{nnz(A)}) $. Cost is higher when evaluating $ g $, due to the fact that matrix $ H $ is not
  perfectly symmetric (it is up to a small epsilon) and not tridiagonal (it has some small
  perturbations). Thus we must use standard QR
  for computing the eigendecomposition, which adds up to a quite large $ O(M N^3)$.
 \end{itemize}

We observe that in Lanczos computing $ g $ it not necessary a drawback when $ A $ is dense or when few
iterations are performed.
It should also be noted that these algorithms suffer from a significant $ O(M N) $ memory cost for storing $ V_M $.


\subsubsection{Why double orthogonalization?}

  From the experimental setting in Section \ref{ssec:ex1}, we observed that the orthogonality of the
  basis $ V $ with Arnoldi was worse than Lanczos basis. Double orthogonalization showed good
  results. One reason could be that the Laplacian matrices that we are considering are badly
  conditioned. The following example motivates this choice:
\begin{example}
    
We consider a Erdős-Rényi graph with $ N = 1000,M=200, p=0.04 $.
The associated Laplacian $ \L $ is terribly conditioned, indeed $ k_2(\L) = 2.365029616834158 \times 10^{17} $:
  \input{assets/tables/orthogonalization.tex}

  \end{example}
  As we can observe in Table \ref{table:orthog}, orthogonalizing two times against the previous
  basis vectors yields to a better orthogonal basis $ V$, while a single Grahm-Schmidt iteration performs worse than standard Lanczos.


\subsubsection{Comparing Lanczos with Arnoldi}

Here we do a short numerical comparison of Lanczos and Arnoldi with double orthogonalization, which
should reflect the cost analysis from Section \ref{ssec:cost}. Algorithm
\ref{alg:arnoldi_double_ortho} reports our implementation of Arnoldi.

\begin{algorithm}[ht!]
  \caption{Arnoldi with double orthogonalization.}
  \label{alg:arnoldi_double_ortho}
  \begin{algorithmic}[1]
  \State \textbf{Input:} Matrix $ A \in R^{N\times N} $, $ s \in \R^N $, $ M \in \N$, function $
    g $, tolerance $ \tau $.
  \State \textbf{Output: } Basis matrix $ V_l = \left[ v_1,\dots,v_l \right] \in R^{N \times l} $
  for $ K_l(A,b) $, upper-Hessenberg matrix $ H \in R^{l \times l} $ where $ l \in
    \{ 1,\dots,M \}$ is the stopping iteration index.

    \State $ v_1 \leftarrow s / \|s\|_2 $
    \State Initialise $ V \leftarrow \mathbf{0}^{N \times M} $ and $ H \leftarrow
    \mathbf{0}^{M \times M} $
    \State $ V_{:, 1} \leftarrow v_1 $
    \For{$ j = 1 \dots M-1 $}
        \State $ w_1 \leftarrow A v_j $
        \State \Comment{First orthogonalization}
        \State $ h_1 \leftarrow V_j^\top w_1 $
        \State $ w_1 \leftarrow w_1 - V_j \cdot h_1 $
        \State $ v_{j+1} \leftarrow w_1 / \|w_1\|_2 $ 
        
        \State \Comment{Second orthogonalization pass}
        \State $ h_2 \leftarrow V_j^\top v_{j+1} $
        \State $ w_2 \leftarrow v_{j+1} - V_j \cdot h_2 $
        \State $ H_{1:j, j} \leftarrow h_1 + h_2 $
        \State $ H_{j+1, j} \leftarrow \|w_1\|_2 \|w_2\|_2 $\\
        
        \If{$ H_{j+1, j} = 0 $}
            \State \textbf{break} \Comment{Arnoldi breakdown}
        \EndIf
        
        \State \Comment{Convergence check}
        \If{$ j > 3 $ \textbf{and} $ j < M $}
            \If{$ \|g_j - g_{j-3}\|_2 < \tau $}
                \State \Return $ V_j, H_{1:j, 1:j} $
            \EndIf
        \EndIf
        
        \State $ v_{j+1} \leftarrow w_2 / \|w_2|_2 $
        \State $ V_{:, j+1} \leftarrow v_{j+1} $
    \EndFor
    
    \State Set elements of $ H $ smaller than machine epsilon to $ 0 $
    \State \Return $ V_{:, 1:j}, H_{1:j, 1:j}$
  \end{algorithmic}
\end{algorithm}

We consider again function the function $ g $ used in the experiments and $ N \in \left\{ 300, 400,
500, \dots, 6000\right\} $, $ M = 200 $ and $ p = 0.04 $. 
In Figure \ref{fig:avl} and the related Table\ref{table:avl} we see different correlation when running
the algorithm with residual break and until $ M $. In fact, when stopping with the residual $
\|g_{l-3} - g_l \|_2 $, in Figure \ref{fig:avl_a}, Lanczos
with full orthogonalization outperforms the other two, while running without residual in Figure
\ref{fig:avl_b}, curves generally match theoretical results, despite some spikes of the full-orthogonalization variant.
One reason for full orthogonalization converging faster than predicted in the complexity analysis,
is probably due the spectrum of $ T_l $ falling outside the definition interval of $ g $, as we will
observe better soon in the experiments in Section \ref{sec:experiments}.

\section{Experiments}
\label{sec:experiments}

In the following section we generate random Erdős-Rényi and Sensor graphs, as defined in Definitions
\ref{def:ER} and \ref{def:Sensor}, to study Lanczos algorithm's accuracy and performance. 

\subsection{Comparing global error with projection error}
\label{ssec:ex1}

We want to numerically verify the approximation in Equation
\eqref{eq:projection} by relating the residuals in Definition \ref{def:errors}, which represent the
algorithmic error and the ground truth $ g(\L) $.
Let's recall some quantities that will be useful in this section: $ N $ is the number of graph
nodes and size of the Laplacian $ \L $, $ M $ is the maximum size of the Krylov subspace, and $ p $
the probability of the graph. 

\begin{experiment}

\end{experiment}

feasible for the small graph that we are considering.

Let's analyse Figure \ref{fig:ex1_estimate}: we immediately observe that the error curves are
closely tight, and that their value is decreasing, which shows that the approximation of $ g $ gains
precision as the basis $ V $ gets larger. We also see how the eigenvalues of $ \L $ and $ T $ influence the steepness and smoothness of the curves. Indeed, when eigenvalues fall outside $
[-\frac{1}{2}, \frac{1}{2}] $, we see that the curves rapidly reach machine precision: this is
unfortunately caused by $ g $ not perfectly matching the Laplacian eigenvalues distribution. We also
observe evident lack of smoothness in the iteration error curve. We believe that due the
irregularity of $ g(t) $ in $ t \in \{  -\frac{1}{2}, \frac{1}{2} \}$, the eigenvalues of $ T $ tend
to jump in and out the interval $ [-\frac{1}{2}, \frac{1}{2}] $, causing spikes. It is also
interesting to visualize graphically in Figure \ref{fig:sigproc} how the signal $ s $ is filtered by
the algorithm on the graphs used in this experiment, which corresponds to applying a low-pass filter.

\subsection{Performance comparison for Erdős-Rényi graphs}
\label{ssec:ex23}

In this experiment, we will study the performance correlation for running Lanczos on Erdős-Rényi
graphs $ \G(N,p) $, comparing time data when only $ N $ varies and when only $ p $ varies. In these
graphs, $ N $ and $ p $ dictate the expected properties of the matrix $ \L $, that is dimension and
sparsity, respectively. In fact, when $ N $ increases, we get expect higher computational cost due
to the increased cost of the matrix-vector multiplication $ \L v $, but when $ p $ increases, $ \L
$ becomes more dense, requiring more floating point operations for the algorithm and so increasing
runtime. This pattern can be seen from Figure \ref{fig:spy}, which plots some of the matrices we used.
We choose $ \delta = 10^{-5} $, as we experimentally observed that smaller thresholds rarely triggered full
orthogonalization. As described in Algorithm \ref{alg:lanczos} our stopping criteria happens when $
\|g_j - g_{j-3}\|_2 < \tau $, at the step $
j$. \footnote{Here
index goes backwards because matrix $ T_l $ would not be defined for $ l>j $ at iteration $ j $!}

\begin{experiment}
In the following experiment, we choose generate two sample datasets, letting $ n = 6 $,:
\begin{enumerate}
  \item $ N $ varies in $ \{ 50,100,\dots,2^n \cdot 50\} $, $ p = 0.04 $.
  \item $ p $ varies in $ \{ 0.01,0.02,\dots,2^n \cdot 0.01 \}$, $ N = 1000$.
\end{enumerate}
We repeat the generation of these graphs $ 250 $ times to avoid graph randomness biasing our results.
 
\end{experiment}

\subsubsection{Run with selective orthogonalization, $ \tau =  10^{-5}$ and $

We observe that when $ N $ or $ p $ increase, the computational time decreases. In the first case,
we know from the theory that when $ N \gg M $, the Krylov subspace is approximated better by
the algorithm, as we can see in Table \ref{table:group_table_2}.
In the second case, as $ p $ grows matrices in Figure \ref{fig:spy} become more dense, and we expect
time becoming higher and the basis loosing orthogonality more easily. Instead, we see the algorithm
converging fast towards the $ \tau $ bound, why?
We believe that this is due the phoenomena we already observed in
Section \ref{ssec:ex1}: the error $ g_{l-3} - g_l $ reaches machine
precision fast as depicted in Figure \ref{fig:ex1_estimate}, because the eigenvalues of the input
matrix $ \L $ are falling outside the domain of $ g $.
To conclude, looking at Figure \ref{sfig:loglog_g} we see a reversed wave-like pattern for times: with shorter time for big $ (N,p) $, and viceversa.
\begin{itemize}

    than for $ p $ for running Lanczos;
  \item When $ (N,p) \in \{ (100,0.2), (200,0.4) \} $, we see the opposite as above; 
  \item For the remaining cases times tend to be shorter and be more similar for $ N $ and $ p $.
\end{itemize}

\section{Conclusions \& future work}

To conclude, we observed how stategic bound and orthogonalization techniques can improve performance
trough early breaks, especially for large and dense matrices. However, due the fact that the funciton
was piecewise continuous on the spectrum values we observed that these bound can start to be
unrelyiable and exit the algorithm too early, when the Krylov subspace may be not well-formed.
We didn't address the high memory cost of $ O(N\cdot M) $ of Lanczos, this could be avoided by
using a two-step implementation, that stores only three vectors and reconstructs the basis when
needed. Further, because full orthogonalization was widely used in the second experiment, we could
implement selective reorthogonalization as described in \cite{demmel1997}, but both methods fall
behind the objective of this course. Another improvement, which we already discussed, would be to
implement a working \texttt{WarpedTranslates} class in Library \cite{pygsp}.
